# 共通アルゴリズム（診断とプラン）

このページは、外来・病棟・手術適応判断に共通して適用できる「診断とプランの同一アルゴリズム」を、再現可能な手順として定義することを目的としています。ここでいうアルゴリズムとは、経験則の列挙ではなく、意思決定の入力（情報）と出力（判断・次アクション）を明示し、**次回外来で何を確かめ、分岐に応じて何をするか**までを含みます。

## 前提（公開に関する制約）

- 症例を特定できる情報（年齢・職業・稀少疾患の組み合わせなどを含む）は記載しません。
- 画像・検査データ・動画は、匿名化と権利関係が担保できる場合に限ります。
- このアルゴリズムは教育的な枠組みであり、個別診療の代替ではありません。  

## 入力→処理→出力（最小単位）

### 入力（Input）

- **主訴**（患者が困っている機能障害を一文で）
- **受傷／発症の基点**（いつ、何を契機に、どの時間経過で）
- **増悪・寛解因子**（荷重、歩行距離、靴、スポーツ、日内変動）
- **赤旗所見**（感染、循環障害、神経障害、腫瘍、急速進行など）

### 処理（Process）

1. **問題表現（Problem representation）**  
   主訴＋基点＋時間軸＋荷重との関係を、医学的な語彙で短文化します。
2. **鑑別の軸を立てる（Differential framing）**  
   例：外傷性／非外傷性、骨性／軟部、炎症性／変性、局所／全身
3. **検証の順序を決める（Test ordering with purpose）**  
   「撮るために撮る」を避け、各検査で否定／支持したい仮説を明示します。
4. **意思決定（Decision）**  
   保存か手術か、あるいは経過観察か。適応条件と禁忌条件を並記します。
5. **プラン（Plan）を分岐で書く（If–Then plan）**  
   次回外来のアクションを含め、**AならB、CならD**を先に定義します。

### 出力（Output）

- **診断（暫定診断を含む）**
- **プラン**（介入＋フォロー＋次回分岐）
- **患者への説明要点**（期待値調整：治療目標、期間、合併症）

## 身体診察の順序（下肢・足部）

系統的な診察は、患者の体位変換を最小限にし、見落としを防ぐために一定の順序で行います。以下は足部・足関節を中心とした下肢診察の標準的な流れです。

### 1. 正面立位（Standing, anterior view）

- 膝のアライメント（genu varum / valgum）
- 下肢全体のアライメント

### 2. 背面立位（Standing, posterior view）

- 脊柱のアライメント（側弯、後弯）
- 膝窩の対称性
- 後足部アライメント（hindfoot varus / valgus）
- アキレス腱の走行

### 3. 歩行観察（Gait）

- 歩行パターン
- 足部外旋（foot progression angle）
- 跛行の有無

### 4. 椅子座位（Sitting）

- 膝Jサイン（patellar tracking）
- 足関節可動域（dorsiflexion / plantarflexion）
- 外側靭帯（ATFL, CFL）
- 内側靭帯（deltoid ligament）
- 後脛骨筋腱（tibialis posterior tendon）
- 腓骨筋腱（peroneal tendons）
- 足部の可動域と疼痛誘発テスト

### 5. 仰臥位（Supine）
）
- 脚長（leg length: true / apparent）
- 股関節可動域（hip ROM）
- 膝可動域（knee ROM）
- 膝安定性テスト（Lachman, drawer, varus/valgus stress）
- 股関節伸展（hip extension / Thomas test）

### 6. 腹臥位（Prone）

- 股関節前捻（hip anteversion / femoral rotation）
- 下腿脚長（tibial length）
- 足部外旋（thigh-foot angle）
