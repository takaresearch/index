# 問診・インターラクションの型（患者との対話）

このページは、患者とのインターラクションを「漏れなく・偏りなく・過剰に侵襲的でなく」進めるための問診テンプレートを定義します。重要なのは、情報収集そのものよりも、(1)患者が語る困りごとを正確に定義し、(2)臨床仮説の検証に必要十分な追加質問へ段階的に移り、(3)意思決定の前提（価値観・生活背景・許容リスク）を同定することです。

## 1. 最初の一文（主訴の同定）

### 目的

主訴を「痛い」ではなく、機能障害として定義します。主訴が曖昧なまま鑑別に入ると、診断は偶然当たってもプランがぶれます。

### テンプレ（コピペ用）

- いちばん困っていることは何ですか（痛み以外も含む）  
- それが起きるのは、どの動作／どの場面ですか（歩行距離、階段、片脚立位、靴、仕事）  
- そのために、何ができなくなりましたか（具体的に）  

## 2. 受傷／発症の基点と時間軸

- いつからですか（起点）  
- きっかけはありますか（外傷／過負荷／靴変更／運動量変化）  
- どのように変化しましたか（急性→改善／慢性進行／波がある）  

## 3. 仮説駆動の追加質問（過不足を減らす）

### 目的

鑑別を列挙するのではなく、「どの仮説を否定／支持したいか」に沿って質問します。

### 典型軸

- 荷重との関係：荷重痛／安静時痛／夜間痛  
- 局在：一点か、広がるか（放散）  
- 腫脹：夕方増悪、圧痕、熱感  
- 神経症状：しびれ、灼熱感、誘発肢位  
- 炎症性徴候：朝のこわばり、対称性、他関節  

## 4. 患者の意思決定前提（価値観と制約）

治療の選択は、解剖学的な正しさだけでなく、患者の価値観と制約の上で成立します。ここを曖昧にすると、説明が「正しいが受け入れられない」形になりやすくなります。

- 生活上の優先順位（仕事・育児・介護・スポーツ）  
- 受け入れ可能なダウンタイム（免荷期間、通院頻度）  
- 望まない結果（瘢痕、可動域制限、疼痛遷延）  
- 手術への態度（恐怖、過去の経験、情報源）  

## 5. 説明の型（期待値調整）

### 目的

患者説明は「情報提供」ではなく、治療目標と観察ポイントを共有し、次回分岐（AならB）を患者に理解可能な形で提示することです。

### テンプレ

- 現時点の見立て：  
- いま一番大事な点：  
- 今日やること：  
- 次回こうだったら（A）→ こうします（B）  
- 逆にこうなら（C）→ こうします（D）  

## 6. 公開運用上の注意

このテンプレートは一般論であり、個別症例の記載は行いません。症例ベースのメモが必要な場合は、公開外（`private/`フォルダなど）に分離し、匿名化・規程遵守を前提とします。
