# 手術記録の型（術直後に仕上げる）

本ページは、手術記録を「術後に思い出す文章」ではなく、「術中の意思決定と再現可能性を担保する記録」として設計し直すためのテンプレートである。術直後に完成させることを原則とし、必要に応じて **自動読み上げ（音声入力）** と **スクライブ** を併用する。

## 0. 原則

- **適応と術式選択の理由**が記録の核である（手順の羅列ではない）。  
- **術中に分岐した点**（予定変更、追加処置、固定変更）を最優先で残す。  
- 「翌日の自分／他者が読んで再現できる」粒度を担保する。  
- 症例同定性のある情報は公開ページに含めない（必要なら `private/` へ）。

## 2. 音声入力＋スクライブの運用

### 推奨ワークフロー

- **術直後5分**：術式と分岐（予定変更・追加手技・固定方針）だけを音声で粗く記録する。  
- **当日中10分**：テンプレに流し込み、見出しを埋めて整形する。  
- **翌日1分**：客観情報（インプラント型番等）を追記し、完了とする。  

### 失敗しやすい点（予防）

- 手順は書けても「なぜその判断か」が抜ける → **適応と分岐**を冒頭に固定する。  
- インプラント情報が散逸する → 末尾に「固定・インプラント」欄を必置する。  

## 3. 公開運用上の注意（必須）

本テンプレートは公開可能であるが、具体症例に依存する記載（日時、施設、特徴的背景、連続した術中写真等）は症例同定性を生むため、公開ページに載せない。症例記録が必要な場合は、公開外（`private/` 等）へ分離し、匿名化と規程遵守を前提とする。
