# 外来の型（プレゼンテーション中心）

本ページは、上級医へのプレゼンテーションを主目的として外来診療を構造化する「外来の型」を定義する。要点は、(1)診断確定前は**主訴中心＋受傷／発症基点**で語り、(2)診断が付いた後は**診断中心＋主訴**に切り替え、(3)プランは次回外来でのアクションを含めて **If–Then で分岐**させる、の3点である。

## 0. 原則

- **診断とプランは常に同じアルゴリズム**（`clinical/algorithm.md`）で回す。  
- **プレゼンテーション = 診断とプラン**である（情報の羅列を目的にしない）。  
- 「次回何をするか」を明示し、**次回で決着がつく設計**に寄せる。  

## 1. 診断確定前：主訴中心で組み立てる

### 目的

診断ラベルの未確定性を隠さず、評価の優先順位（何を否定／支持するか）を上級医と共有する。

### プレゼン骨格（コピペ用）

---

- 主訴：  
- 受傷／発症基点：  
- 経過：  
- 主要身体所見：  
- 画像（あれば要点のみ）：  
- 鑑別（上位3）：  
- 現時点の仮説：  
- 本日のアクション（検査／固定／薬物／リハ等）：  
- 次回分岐：  
  - If A（例：疼痛持続／特定所見陽性）→ Then B（追加検査／介入）  
  - If C（例：改善傾向）→ Then D（負荷調整／経過観察）  

---

## 2. 診断確定後：診断中心で組み立てる

### 目的

「診断に対して、なぜその治療戦略か」を短時間で説明可能にし、次の一手（手術適応判断を含む）を明確化する。

### プレゼン骨格（コピペ用）

---

- 診断：  
- 主訴（機能障害の要約）：  
- 重症度／ステージ（あれば）：  
- 治療目標（短期／中期）：  
- プラン：  
  - 本日：  
  - 次回：  
  - If A → Then B  
  - If C → Then D  
- 手術を検討する場合：適応条件／回避条件（禁忌）  

---

## 3. 次回アクション設計（AならB、CならD）

次回分岐は「都度考える」から「事前に定義する」へ移すことで、診療の一貫性と学習効率が上がる。典型的な分岐は以下である。

- **症状**: 改善／不変／増悪  
- **身体所見**: 誘発テスト陽性の持続／消失  
- **画像**: 追加撮像の要否（仮説検証のための追加）  
- **治療反応**: 固定・薬物・注射・リハへの反応  

## 4. 実装メモ（運用上の注意）

- 個別症例の固有情報を記載しない。必要な場合は公開外（`private/`等）へ分離する。  
- “上級医に相談するための情報”と“患者説明に必要な情報”は重なるが同一ではない。前者は仮説と分岐設計、後者は期待値調整（治療期間と見通し）に重心がある。
